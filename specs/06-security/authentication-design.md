# Authentication Design

## Local Mode

- Stdio sessions inherit trust from the local client launch path.
- No separate end-user login is required for local workstation use.
- The server still logs client metadata when available.

## Hosted HTTP Mode

- Require bearer-token authentication validated by a token verifier.
- Prefer OAuth 2.1-compatible authorization infrastructure where enterprise deployment requires it.
- Reject missing or invalid tokens before tool invocation.

## Blender Bridge

- The controller bridge authenticates every request with a short-lived shared secret.
- Secrets are generated by the MCP server and handed to the controller out-of-band during startup.
- The controller invalidates old secrets on restart.
