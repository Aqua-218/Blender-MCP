# Compliance Considerations

## Default Position

The core product is not inherently a regulated-data processing platform. It becomes compliance-relevant only when operators use it with regulated content or deploy it in regulated environments.

## Baseline Compliance-Friendly Measures

- Auditability through immutable operation logs
- Configurable retention policies
- Access control by role and transport mode
- Path restrictions to prevent accidental overreach

## Enterprise Extension Path

For enterprise or regulated environments, add:

- Centralized identity integration for hosted HTTP mode
- Centralized log shipping
- Formal retention and deletion policies
- Documented export review workflows
